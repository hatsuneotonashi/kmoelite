# 路线图

本路线图描述公开项目方向，不承诺发布日期。kmoelite 当前处于 Alpha / 开发预览阶段。

## 产品方向

- 建立轻量非官方 KMOE 在线漫画阅读体验。
- 优先解决手持设备存储成本高、漫画缓存容易膨胀的问题。
- 默认路径是“点开一本看一本”，通过临时 Reader cache 保持高清阅读，阅读结束后按策略清理。
- 永久下载、Library 和文件打开能力保留为高级/兼容能力，不作为普通阅读默认路径。
- 不加入绕过登录、会员、配额、验证、反滥用或版权限制的能力。

## 近期

- 保持公开 source tree 干净、可复现、无敏感信息。
- 把 README、docs、状态文档维持为 Alpha / 开发预览口径。
- 强化 Reader cache policy：读完清理、章节切换清理、存储压力清理和可解释的空间占用。
- 继续拆分较大的页面、native 模块和 CSS 文件，但不改变产品行为。
- 把平台状态、release 限制和验证记录保持为可证明事实。

## 平台计划

- iPhone：触控优先、安全区、单手操作、低存储占用。
- iPad：分栏/侧栏、横竖屏、高清大屏阅读。
- Android 手机：计划支持，需完成 Android runtime、打包和真机验证。
- Android 平板：计划支持，需完成平板布局和真机验证。
- Windows：计划支持，需完成真实安装、卸载、open/reveal 和签名验证。
- macOS：当前主要开发平台，公开二进制发布仍需签名、公证和干净机器验证。
- Apple TV：后续研究方向，需先设计遥控器输入、横屏阅读和缓存策略。
- Android TV：后续研究方向，需先设计遥控器输入、横屏阅读和缓存策略。

## 发布准备

- 完成 macOS 签名、公证、stapling 和干净机器安装/打开验证。
- 完成 Windows installer、uninstall、SmartScreen、open file、reveal folder 验证。
- 完成 iPhone/iPad 签名真机验证。
- Android 平台实现后补充 Android 真机和打包验证。
- 保持 release manifest 可复现且脱敏。

## Reader 体验

- 强化单页、双页、连续、RTL/LTR、缩放、裁切、旋转、缩略图和手势体验。
- 保持漫画图片本身为 Reader 主体，减少 chrome 干扰。
- 支持超高清阅读目标，但不以长期保存超大本地文件作为默认方式。
- MOBI 在实现真实 parser 前仍为 file-only。

## 合规边界

- 保持项目非官方表述。
- 不公开传播站点内部授权接口细节。
- 真实站点检查保持显式、低频、脱敏、默认关闭。
